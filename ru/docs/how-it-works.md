# Как это работает — AI Workspace OS

Методология, лежащая в основе системы.

---

## Ключевая проблема

AI-инструменты мощные — но большинство людей используют их как поисковик:
один вопрос, один ответ, без памяти, без структуры.

Каждая сессия начинается с нуля. Агент не знает:
- Кто вы
- Над каким проектом работаете
- Что решили на прошлой неделе
- В каком формате вам нужен результат

Вы повторяете одно и то же. Получаете шаблонные ответы. Тратите время на правки вместо работы.

---

## Решение: структурированный контекст

AI Workspace OS даёт агенту постоянную рабочую среду.

Три уровня контекста:

```
УРОВЕНЬ 1: Глобальный (кто вы)
  → about-me.md + global-rules.md
  → Загружается каждую сессию
  → Определяет личность, предпочтения, стандарты результата

УРОВЕНЬ 2: Проектный (над чем работаете)
  → context/context.md + project-rules.md
  → Загружается для конкретного проекта
  → Определяет цели, предысторию, ограничения, историю

УРОВЕНЬ 3: Задача (что делать сейчас)
  → task-brief.md или session-start.md
  → Пишется для каждой сессии
  → Определяет конкретную задачу на сегодня
```

Каждый уровень — markdown-файл, который вы пишете один раз и используете повторно.

---

## Поток сессии

```
Начало сессии
    ↓
Вставить prompt из session-start.md
    ↓
Агент читает: about-me → global-rules → context → log
    ↓
Агент подтверждает понимание
    ↓
Вы описываете задачу (или используете task-brief.md)
    ↓
Агент работает
    ↓
Агент записывает результат в logs/log.md
    ↓
Конец сессии
```

Лог становится поисковой историей всех сессий.
В следующей сессии агент читает лог и знает, что уже было сделано.

---

## Почему это работает

**Для нетехнических пользователей:**
Без кода, без API, без автоматизации. Только текстовые файлы и копипаст prompt-ов.
Работает с любым AI-инструментом, который принимает текстовый ввод.

**Для стабильного качества результата:**
Агент всегда работает по одним и тем же поведенческим правилам.
Не нужно каждую сессию заново объяснять предпочтения.

**Для непрерывности проектов:**
Лог + файлы контекста работают как постоянная память.
Даже если вы смените AI-инструмент — контекст переезжает с вами.

**Для команд:**
Файлы контекста можно расшарить. Одна команда → одна общая папка `global/` → одинаковое поведение AI у всех.

## Принципы проектирования

**1. Только текстовые файлы.**
Нет баз данных, нет специального софта. Всё — `.md`-файлы, которые можно открыть в любом редакторе.

**2. Читаемо для человека.**
Файлы написаны так, чтобы их могли читать и люди, и AI. Можно редактировать вручную в любой момент.

**3. Постепенная детализация.**
Начните с `about-me.md` и `global-rules.md`. Добавляйте проекты и шаблоны по мере необходимости.
Система работает при любом уровне заполненности.

**4. Не привязан к AI.**
Работает с Claude, ChatGPT, Gemini или любой моделью, принимающей текстовый prompt.

**5. Готовые настройки по умолчанию, легко переопределить.**
Базовые правила покрывают 80% профессиональных сценариев.
Переопределяйте что угодно в `project-rules.md`, не трогая глобальные настройки.

---

## Чем это не является

- Не инструмент автоматизации (скрипты не запускаются автоматически)
- Не интеграционная платформа (нет API-подключений)
- Не база данных (всё — обычный текст)
- Не готовый продукт (это фреймворк — вы настраиваете под себя)
