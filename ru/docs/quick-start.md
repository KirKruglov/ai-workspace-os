# Быстрый старт — AI Workspace OS

Запуск за 5 минут.

---

## Что это

AI Workspace OS — это система папок + набор шаблонов, которые превращают любой AI-чат
(Claude, ChatGPT и др.) в структурированного рабочего ассистента.

Вместо того чтобы начинать каждую сессию с нуля, вы даёте AI постоянный контекст:
кто вы, как работаете, над каким проектом, что было сделано раньше.

**Результат:** лучше выход, меньше повторений, быстрее сессии.

---

## Шаг 1 — Скачайте

Нажмите **Code → Download ZIP** на странице GitHub-репозитория, затем распакуйте папку.

Или через Git:
```bash
git clone https://github.com/KirKruglov/ai-workspace-os.git
```

---

## Шаг 2 — Заполните профиль

Откройте `global/about-me.md` и заполните:
- Ваше имя, роль и компанию
- Текущие приоритеты
- Предпочтения по коммуникации

Откройте `global/global-rules.md` и настройте:
- Язык (English или Русский по умолчанию)
- Предпочтения по формату результата
- Любые правила, которые хотите изменить

> Эти два файла — фундамент. Чем лучше вы их заполните, тем лучше будет работать AI.

---

## Шаг 3 — Создайте первый проект

1. Скопируйте папку `projects/_template/`
2. Переименуйте под ваш проект — напр. `projects/product-launch/`
3. Откройте `projects/product-launch/context/context.md`
4. Заполните: название проекта, цели, предысторию, ограничения

---

## Шаг 4 — Начните первую сессию

Откройте `templates/session-start.md`, скопируйте prompt, заполните пропуски и вставьте в Claude или ChatGPT.

Агент:
1. Прочитает глобальный контекст
2. Прочитает контекст проекта
3. Подтвердит понимание задачи
4. Приступит к работе

---

## Шаг 5 — Поставьте задачу

Используйте `templates/task-brief.md` для структурирования задачи.
Хорошо написанный бриф занимает 2-3 минуты и экономит 3-5 сообщений переписки.

---

## Структура папок

```
ai-workspace-os/
├── global/                    ← Кто вы + как ведёт себя агент
│   ├── about-me.md
│   └── global-rules.md
├── projects/
│   └── your-project/          ← По одной папке на проект
│       ├── context/           ← Контекст, цели, правила
│       ├── input/             ← Исходные материалы (документы, данные)
│       ├── output/            ← Финальные результаты
│       └── logs/              ← История сессий
├── templates/                 ← Готовые prompt-шаблоны
│   ├── session-start.md
│   ├── task-brief.md
│   └── agent-instructions.md
└── docs/                      ← Вы здесь
```

## Советы

**Держите файлы контекста короткими и актуальными.**
Если `context.md` устарел — агент будет работать с неактуальной информацией.

**Пусть агент логирует работу.**
В конце каждой сессии попросите: *"Запиши, что мы сделали, в logs/log.md"*.
Так накапливается история, на которую можно ссылаться позже.

**Одна папка проекта на одну тему.**
Не смешивайте несвязанные задачи в одной папке проекта. Создавайте отдельные.

**Используйте бриф для сложных задач.**
Для простых вопросов бриф не нужен. Для всего, что требует больше одного сообщения, — используйте `task-brief.md`.

---

## Решение проблем

**Агент игнорирует файлы контекста.**
→ Убедитесь, что вставляете содержимое файла прямо в чат, а не просто упоминаете имя файла.
Некоторые AI-инструменты не имеют доступа к файлам — вставляйте содержимое вручную.

**Качество результата низкое.**
→ Проверьте `global/global-rules.md` — добавьте более конкретные требования к формату.
→ Заполните `global/about-me.md` подробнее.

**Не знаю, как структурировать новый проект.**
→ Начните с заполнения только `context.md` в папке проекта. Этого достаточно для старта.
