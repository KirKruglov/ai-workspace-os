# Task Brief Template

> **How to use:**
> Fill out this template before giving a task to your AI agent.
> A complete brief = better output, fewer back-and-forth iterations.
> Delete sections that don't apply.

---

## Task Brief

**Date:** [YYYY-MM-DD]
**Project:** [Project name — or "standalone" if not tied to a project]
**Priority:** High / Medium / Low

---

### 1. Task

> What needs to be done? One clear sentence.

[Write the task here]

---

### 2. Context

> Why does this matter? What should the agent know to do this well?

**Background:**
[Relevant context — what led to this task, what's happening in the project]

**Key information:**
- [Fact, constraint, or data point the agent needs]
- [Another relevant detail]
- [Add more as needed]

**Related files or documents:**
- `input/[filename]` — [what it contains]

---

### 3. Deliverable

> What exactly should the output be?

**Type:** [Document / Table / Analysis / Draft / Plan / Summary / Other]
**Format:** [.md / .xlsx / .docx / inline response]
**Length:** [Approximate — e.g. "1 page", "5-7 bullet points", "full analysis"]
**Audience:** [Who will read or use this output]

---

### 4. Acceptance Criteria

> How will you know the output is good?

- [ ] [Criterion 1 — e.g. "Includes competitor comparison table"]
- [ ] [Criterion 2 — e.g. "Written for non-technical audience"]
- [ ] [Criterion 3]

---

### 5. Constraints

> What should the agent avoid or be careful about?

- [Constraint — e.g. "Don't include pricing data, it's confidential"]
- [Constraint — e.g. "Tone must be formal, this goes to the board"]
- [Or write "None"]

---

### 6. Deadline (optional)

[When do you need this by — or leave blank]

## Example (filled in)

**Date:** 2026-03-11
**Project:** Q2 Product Launch
**Priority:** High

### 1. Task
Write a one-page executive summary of our product positioning for the Q2 launch.

### 2. Context
**Background:** We're launching a new B2B analytics feature in April. The summary will be
used in the board deck and as context for the sales team.

**Key information:**
- Target segment: mid-market SaaS companies (50–500 employees)
- Key differentiator: real-time data vs. competitors' 24h lag
- Current market leader: Mixpanel

**Related files:**
- `input/positioning-notes_2026-02-20.md` — raw notes from strategy session

### 3. Deliverable
**Type:** Executive summary
**Format:** .md
**Length:** 1 page, max 400 words
**Audience:** Board of directors + VP Sales

### 4. Acceptance Criteria
- [ ] Covers: problem, solution, differentiation, market opportunity
- [ ] No jargon — readable by non-technical board members
- [ ] Ends with a clear call to action

### 5. Constraints
- Don't mention specific customer names
- Don't include pricing
